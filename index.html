El significado detrás del nombre "Git"

Linus Torvalds, creador de Git, nombró la herramienta como un juego de palabras. En inglés británico, "git" es un término coloquial que significa algo así como "idiota". Torvalds dijo en broma que él mismo es un "git egoísta" por crear esta herramienta.

Un repositorio dentro de otro

En Git puedes tener un repositorio dentro de otro usando submódulos. Esto se utiliza para proyectos que dependen de otros repositorios externos, permitiendo gestionarlos como parte del principal.

El compromiso con la velocidad

Git fue creado principalmente para ser rápido. De hecho, Linus diseñó la herramienta porque no estaba satisfecho con la velocidad de otros sistemas de control de versiones como BitKeeper, que usaba para el kernel de Linux.

Git no almacena archivos de forma literal

A diferencia de otros sistemas de control de versiones, Git no almacena los archivos en cada commit como copias completas. En su lugar, usa un sistema basado en snapshots, donde solo guarda las diferencias y un índice del estado completo en cada commit. Esto ahorra mucho espacio.
