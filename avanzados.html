Deshacer un commit(perdiendo cambios):

Con reset --soft puedes deshcacer el ultimo commit realizado, pero mantiene los cambios en el área de staging. En consola se ejecuta con el comando: git reset --soft HEAD~1


Hecho por Amanda Correas

Deshacer un commit(manteniendo cambios):

Con reset --soft puedes borrar el ultimo commit realizado y los cambios asocialdos a él. En consola se ejecuta con el comando: git reset --hard HEAD~1

Hecho por Amanda Correas

Branch:

Con branch puedes crear ramas, hacieno mas caminos que solo el principal, con -d puedes eliminar ramas, con -r puedes ver las raas remotas. En consola se ejecuta con el comando: git branch "nombre de la rama"


Hecho por Amanda Correas

Tag:

Con tag puedes crear una etiqueta. En consola se ejecuta con el comando: git tag -a v1.0 -m "Versión 1.0"

Ademas si ponemos el comando: git tag veremos las etiquetas existentes


Hecho por Amanda Correas

stash:

Con stash puedes guardas cambia que aun no se han corfirmado. En consola se ejecuta con el comando: git merge stash

Hay varios parametros, como Apply para recuperar cambios guardados o drop para eliminar un stash


Hecho por Amanda Correas

Rebase:

Con rebase puedes reaplicar commits de una rama sobre otra, creando un historia mas limpio. En consola se ejecuta con el comando: git rebase "rama base"


Hecho por Amanda Correas

Submodule:

Con submodule puedes añadir submodulos apunta a una confirmación específica en otro repositorio externo. En consola se ejecuta con el comando: git submodule add "URL"


Hecho por Amanda Correas

Blame:

Con Blame puedes monstrar línea por línea quién cambió cada parte del archivo. En consola se ejecuta con el comando: git blame "archivo"


Hecho por Amanda Correas
